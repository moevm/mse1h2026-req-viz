## Протокол встречи
1. Для решения поставленной задачи использовать клиент-серверную архитектуру. Бэкенд реализовать на Python с использованием FastAPI. Пользовательский интерфейс реализовать с использованием Streamlit. 
2. Пользователь, при первом заходе на страницу проекта видит поисковую строку, куда вводит название технологии. По запросу пользователя отображается граф.
3. Отображаемый граф содержит отношения: "Разработано компанией", "Альтернатива для", "Используется с", "Зависит от", "Используется лицензия"; Перечень отношений должен иметь возможность быть расширен
4. При вводе пользователем запроса, не связанным с темой IT (конкретных технологий) построение графа не выполняется
5. Сформированный граф сохраняется в БД, при повторном запросе данной технологии граф строится из БД
6. Отображаемый граф имеет возможность настройки отображаемых отношений, а также глубины их отображения
