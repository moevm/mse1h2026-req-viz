# mse-template

## Установка и запуск


### Требования

Перед запуском убедитесь, что установлены:

1. Docker
2. Docker Compose
3. Git

Проверить установку:

```bash
docker --version
docker compose version
git --version
```

Docker используется для контейнеризации приложения и запуска всех сервисов одной командой.

---

### Клонирование репозитория

```bash
git clone https://github.com/moevm/mse1h2026-req-viz.git
cd mse1h2026-req-viz
```

---

### Запуск приложения

Основной способ запуска:

```bash
docker compose up --build
```

Что происходит:

- собираются Docker-образы
- запускаются контейнеры
- поднимаются сервисы приложения

Docker позволяет запускать приложение одинаково на разных машинах без проблем с зависимостями.

---

### Доступ к приложению

После запуска приложение будет доступно по адресу:

```
http://localhost:8501
```

---

### Остановка приложения

Чтобы остановить контейнер:

```bash
docker compose down
```

Удалить контейнеры и связанные данные:

```bash
docker compose down -v
```

## Проверка работоспособности
Инструкции по проверке работоспособности проекта (основной функциональности и результатов).

1. Проверка запуска сервисов
Все сервисы работают:

```bash
docker compose ps
```

Ожидаемый вывод:
```
NAME                      STATUS              PORTS
ecosystem-backend         Up                  0.0.0.0:8000->8000/tcp
ecosystem-frontend        Up                  0.0.0.0:8501->8501/tcp
ecosystem-neo4j           Up                  0.0.0.0:7474->7474/tcp, 0.0.0.0:7687->7687/tcp
```

2. Проверка Backend API
Health Check (здоровье системы):
```bash
curl http://localhost:8000/api/health
```
Ожидаемый ответ:
```
{"status":"ok","connected":true}
```

3. Проверка Frontend (Streamlit)
Открыть веб-интерфейс:
```
http://localhost:8501
```

4. Проверка Neo4j (база данных)
Подключение к Neo4j Browser:
```
http://localhost:7474
```
Ввойдите с помощью логина и пароля (Логин: neo4j, Пароль: test1234)

### Настройка шаблона промпта для генерации отчетов

Приложение использует шаблон промпта для генерации аналитических отчетов с помощью Ollama LLM. Шаблон находится в файле:

```
ecosystem_analyzer/templates/report_prompt.md
```

#### Изменение шаблона промпта

Вы можете настроить шаблон промпта для изменения формата и содержания генерируемых отчетов:

1. **Редактирование шаблона по умолчанию:**
   ```bash
   nano ecosystem_analyzer/templates/report_prompt.md
   ```

2. **Создание собственного шаблона:**
   ```bash
   cp ecosystem_analyzer/templates/report_prompt.md ecosystem_analyzer/templates/my_custom_prompt.md
   ```

   Затем отредактируйте новый файл и укажите его имя через переменную окружения:
   ```env
   PROMPT_TEMPLATE=my_custom_prompt.md
   ```

#### Структура шаблона

Шаблон использует плейсхолдеры для подстановки данных:
- `{tech_name}` - название технологии
- `{graph_data}` - структура графа в формате JSON

Пример секции в шаблоне:
```markdown
## Обзор технологии "{tech_name}"
Краткое описание технологии и её роль в экосистеме.

Структура графа:
{graph_data}
```
